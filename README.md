# Лабораторная работа №1 — Observer (JavaFX)

Учебный JavaFX-проект, демонстрирующий применение паттерна проектирования
**Observer (Наблюдатель)** для реактивного обновления пользовательского интерфейса
на основе изменения состояния сервера времени.

Приложение позволяет:
- запускать и останавливать сервер времени;
- отображать прошедшее время;
- визуально реагировать на события с заданной периодичностью;
- отслеживать состояние сервера.

---

## Возможности

- Сервер времени с шагом обновления 1 секунда.
- Отображение количества прошедших секунд.
- Световой индикатор, меняющий цвет каждые 3 секунды.
- Анимация прямоугольника каждые 5 секунд.
- Отображение состояния сервера (активен / неактивен).
- Запуск и остановка сервера из интерфейса.

---

## Архитектура и логика работы

- `TimeServer` реализует интерфейс `Subject` и хранит текущее состояние времени.
- Наблюдатели реализуют интерфейс `IObserver` и подписываются на сервер.
- Каждую секунду сервер:
    1. увеличивает значение времени;
    2. уведомляет всех наблюдателей.
- Каждый наблюдатель реагирует на изменение состояния по своей логике.
- Обновление UI выполняется через `Platform.runLater()`.

---

## Реализации наблюдателей

- `TimeLabelObserver` — обновление текстовой метки времени.
- `SignalObserver` — смена цвета круга каждые 3 секунды.
- `AnimationObserver` — запуск анимации каждые 5 секунд.
- `ServerStateObserver` — отображение состояния сервера.

# Лабораторная работа №2 — Chain of Responsibility (JavaFX)

JavaFX-приложение, демонстрирующее использование паттерна  
**Chain of Responsibility (Цепочка обязанностей)** в игре «Угадай число».

Пользователь должен угадать случайное число от 1 до 10 за ограниченное
количество попыток.

---

## Возможности

- Генерация случайного числа от 1 до 10
- Ограниченное количество попыток
- Проверка диапазона вводимого значения
- Последовательная обработка логики через цепочку обработчиков
- Определение состояния игры (победа / поражение / ошибка)

---

## Архитектура

Логика обработки ввода реализована через цепочку обработчиков:

RangeCheckHandler → AttemptsCheckHandler → GuessCheckHandler → GameOverHandler

Каждый обработчик отвечает за одну проверку и при необходимости
прерывает дальнейшую обработку.

---

## Основные классы

- `GameModel` — хранит состояние игры, количество попыток и загаданное число
- `Handler` — абстрактный базовый класс цепочки
- `GameState` — перечисление состояний игры:
  `NONE`, `OUT_OF_RANGE`, `WIN`, `LOSE`, `WRONG`
